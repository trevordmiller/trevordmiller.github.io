Code principles

  • Tradeoffs. Encourage acknowledging pros and cons so that developer interactions are respectful, open-minded, and pragmatic.
  • Shared. Encourage re-using existing solutions so that developers can focus on project-specific problems.
  • Reliable. Encourage stable, semantically versioned, safe, standard options so that maintenance is simplified.
  • Encapsulated. Encourage portable, deterministic, explicit, declarative solutions so that code complexity is decreased.
  • Automated. Encourage static analysis, tests, documentation, scripts, and services so that manual workflows are replaced with automation.
  • Efficient. Encourage minimal, fast, lightweight options so that performance is improved.
